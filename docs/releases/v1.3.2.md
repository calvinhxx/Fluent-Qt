Fluent-Qt 1.3.2 tightens the library package boundary and makes the Gallery clearer and smoother to use as component documentation.

## Cleaner library package

- Installed development packages and the library-only source archive now use an explicit public-header allowlist. New implementation headers no longer become public API merely because of their source directory, while existing 1.x include paths remain available.
- Source-package and installed-package consumers now receive the same curated FluentQt header set. No application-side API migration is required.

## More useful Gallery

- Component pages now show the public header, qualified C++ type, and `FluentQt::FluentQt` CMake target before their live examples.
- Incomplete placeholder pages have been removed so missing documentation is detected instead of being presented as finished content.
- The project website now leads with the UI library, real Gallery screenshots, component families, the three supported CMake integration methods, and platform downloads.

## Smoother navigation

- Switching between resident pages now stays constant-time as more Gallery pages are opened.
- Cold-page construction avoids repeated theme, style, and layout work. Shimmer feedback remains immediate, while stale requests from rapid navigation are discarded before they can replace the current page.
- Debug builds use a short cold-page debounce, making rapid navigation easier to inspect on Windows without changing Release behavior.

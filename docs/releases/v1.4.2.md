Fluent-Qt 1.4.2 strengthens the reusable UI foundation, makes text-field
behavior more predictable, and adds deeper automated validation for component
contracts.

## Foundation reliability

- AnchorLayout now treats right and bottom anchors as exact visual boundaries,
  preserves fill geometry for widgets with fixed size policies, and safely
  handles spacers and nested layout items.
- QMLPlus state transitions reject unknown states, restore properties omitted
  by the next state, and safely discard state data when target objects are
  destroyed.
- Oversized overlay cards now use a stable available origin instead of
  producing invalid clamp bounds, while the zero-elevation token produces no
  visible shadow in either theme.
- Bundled resources and the Fluent icon catalog can be accessed safely before
  `QApplication` is created; font initialization remains retryable once the GUI
  application exists.

## Text editing and labels

- Label keeps direct, inherited, and Qt meta-property text access coherent while
  preserving application-owned style-sheet declarations when Fluent color roles
  change.
- TextEdit forwards focus to its inner editor, recomputes height after
  width-driven wrapping, preserves undo history when text is reapplied, and
  keeps visible-line bounds ordered.
- TextEdit context menus and FlipView wheel-transition tests are more stable
  across cold starts and desktop platforms.

## Validation and maintenance

- Component contracts now have a focused CTest target and labels for foundation,
  resource-startup, Label, and TextEdit behavior.
- A Linux ASan/UBSan lane checks the active contracts for memory-safety and
  undefined-behavior regressions.
- Weekly full validation compiles every registered Qt/GTest target, improving
  coverage for tests outside the fast pull-request matrix.

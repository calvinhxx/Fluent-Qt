Fluent-Qt 1.4.1 sharpens controls at fractional display scales, keeps dialogs
inside their owning window, and rounds out native desktop behavior across the
Gallery.

## High-DPI rendering and icon clarity

- Shared device-pixel alignment now keeps Fluent surfaces, borders, and glyphs
  crisp at fractional scales and at high-density settings up to 300%.
- Controls use a common icon painting path, improving chevron placement,
  pressed-state fading, and icon consistency throughout inputs, navigation,
  collections, and status surfaces.
- Gallery navigation glyphs and SplitButton layouts have been tightened,
  including correct centering for icon-only buttons and safe spacing between
  labels and the chevron segment.
- Gallery now follows the operating system and Qt display scale directly. The
  previous app-level scale multiplier and restart flow have been removed to
  avoid double scaling and inconsistent window placement.

## Dialogs and desktop behavior

- Dialog and CoachMark are hosted as same-window overlays instead of separate
  native transient windows, preventing owner-content promotion and frozen input
  on macOS while preserving modal behavior.
- macOS Dock Quit and Command-Q now exit correctly even when close-to-tray
  behavior is enabled.
- Custom window chrome can provide localized caption-button tooltips for
  minimize, maximize, restore, and close actions.

## Application integration and Gallery polish

- DatePicker and TimePicker expose locale control, while hard-coded English
  placeholders and titles have been removed so applications own user-facing
  copy.
- The Foundation overview now includes a dedicated Spacing tile, and Gallery
  icon placement has been aligned across navigation and category surfaces.
- The project site has refreshed visuals and tighter quick-start guidance.
